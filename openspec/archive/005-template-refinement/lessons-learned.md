## 3. 可复用模式

### 模式 1: 模板内嵌规则模式

**来源**: 005-template-refinement

**模式描述**:
将关键规则（如状态同步）直接嵌入模板文件顶部，而非分散在单独的规范文档中。好处：
1. AI/用户读取模板时自动获取规则
2. 减少查找成本
3. 规则与模板结构紧密关联

**适用场景**:
- 模板有复杂填写规则
- 规则需要与模板结构同步更新
- 新用户/AI 需要快速上手

**执行步骤**:
```
1. 识别模板的关键填写规则
2. 在模板顶部添加"使用说明"或"规则说明"章节
3. 用表格或列表形式呈现规则
4. 添加填写示例强化理解
```

---

### 模式 2: 向后兼容验证模式

**来源**: 005-template-refinement

**模式描述**:
修改模板或规范时，必须验证现有格式仍然有效。核心原则：
1. 新增而非删除字段
2. 不强制变更现有格式
3. 用已归档 feature 进行兼容性测试

**适用场景**:
- 修改核心模板或规范
- 项目有已完成的 feature
- 需确保不破坏现有工作流

**执行步骤**:
```
1. 列出要修改的字段/结构
2. 检查已归档 feature 的实际格式
3. 确保修改是新增而非删除
4. 在 test-summary 中记录兼容性验证结果
```

## 4. 技术决策

| 决策 | 理由 | 替代方案 |
|------|------|----------|
| **状态同步规则嵌入模板而非单独文档** | AI 读取模板时自动获取，减少查找 | 在 conventions.md 维护（分散，查找成本高） |
| **spec.template.md 使用数字编号章节** | 与已归档 spec 格式一致 | 无编号（可能不一致） |
| **扩展字段指南放在 runstate.template.md** | 用户/AI 需要时直接查看 | 单独文档（分散） |
