## 问题详情

### 问题 1: AI 会话中断后如何恢复？

**来源**: 001-governance-foundation

**问题描述**:
AI 会话中断后，上下文丢失，无法继续之前的工作。

**影响**: 高 - 可能导致工作丢失

**解决方案**:
1. 读取 `AGENTS.md` 了解项目结构
2. 读取当前 feature 的 `runstate.md`
3. 根据"下一步动作"继续执行

**预防措施**:
- 每个 feature 都维护 runstate.md
- 每完成一个任务就更新状态
- 记录明确的"下一步动作"

---

### 问题 2: Spec 与实现不一致怎么办？

**来源**: 001-governance-foundation

**问题描述**:
实现过程中发现 spec 描述与实际情况不符，需要调整。

**影响**: 高 - 可能导致返工

**解决方案**:
1. 停止实现
2. 更新 spec 并说明原因
3. 获得用户确认
4. 继续实现

**预防措施**:
- spec 阶段充分评审
- 明确假设和约束
- 预留调整空间

---

### 问题 3: skill 工具无法调用自定义 skill

**来源**: 002-workflow-dogfooding

**问题描述**:
OpenCode skill 工具只支持内置 skill，无法调用项目自定义的 feature-start, feature-spec 等命令。

**影响**: 高 - 阻塞自动化执行

**临时解决方案**:
- 手动读取 SKILL.md 文件并按步骤执行
- 将 SKILL.md 作为执行指南使用

**长期解决方案**:
- 创建 skill 调用包装器（指南 + 脚本）
- 在 004 feature 中实现

**状态**: 待 004 feature 解决

## 问题分类

### 按影响程度

| 级别 | 定义 | 处理优先级 |
|------|------|------------|
| 高 | 阻塞核心流程 | 立即处理 |
| 中 | 影响效率但不阻塞 | 计划处理 |
| 低 | 影响体验 | 可延后 |

### 按问题类型

| 类型 | 示例 | 典型解决方式 |
|------|------|--------------|
| 工具限制 | skill 调用问题 | 替代方案或等待支持 |
| 流程缺口 | 知识文件缺失 | 补充缺失部分 |
| 规范模糊 | 状态同步规则 | 明确规范 |

## 问题处理流程

```
发现问题 → 记录到 runstate.阻塞点 → 分类评估 → 
确定优先级 → 分配到 feature → 解决并验证 → 沉淀到此文件
```
